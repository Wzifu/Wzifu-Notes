线性回归 最小二乘

通常用房价模型举例

  • input x∈R2 (比如为房价的特征，年龄，产权，位置等等)

  • Prediction: f(x)=xTw (e.g. housing price)

比如

f(X)=w1x1+w2x2+⋯+wdxd

w1就是我们一直说的权重了,x向量和wi是一样的维度。

  • Training inputs: x1,⋯,xn∈Rd.

  • Traing targets: y1,⋯,yn∈R

  • Loss function : L(w)=∑i=1n12(xiTw−yi)2

x是向量，xiTw是预测的房价，yi，两个越接近越好。有n个房子，所以是n的求和

  • Least squares regression(最小二乘回归）: w∗=minwL(w)

Parallel Gradient Descent for Least Squares

即最小二乘法的并行梯度下降法

  • Parallel Gradient Descent

    Example: GD for least squares regression model

    Loss function : L(w)=∑i=1n12(xiTw−yi)2 为了使得这个最小

  • Gradient:

    g(w)=∂L(w)∂w=∑i=1n∂12(xiTw−yi)2∂w=∑i=1n(xiTw−yi)xi

梯度就是那个导数，梯度对向量的求导，也是向量。

其中主要w特征，和并行计算有关，需要不停的传来传去，

其中: g(w)=∑i=1ngi(w) ,where gi(w)=(xiTw−yi)xi

因此有n个训练样本，gi每一个样本只与一个xi,yi有关，因此可以把样本给到多个机器上，每个机器只需要用本地的样本去算本地的gi

  • Gradient descent:

    wt+1=wt−α⋅g(wt).

我们希望找到一个w 使得L(w)的值越小越好，所以可以使用梯度下降，沿着梯度相反的方向走。即在一个w上，沿着梯度方向g(w)走很小一步，这个函数值L（w)肯定是上升的，如果沿着相反的走，那么了L(W)肯定会下降。 -g(w)就是按照梯度下降的方法走。其中α叫做步长或者学习率。那么最后L（w)最后就会收敛，达到最优。

代码每一轮迭代都要算一个梯度。每一个样本算一个gi,全加起来。同时和x,y的维度也有关系，维度越高，计算越麻烦。

所以梯度计算就是影响算法的瓶颈，这部分越快，整体部分越快

  • 用两个处理器加速计算 Parallel GD using 2 processors

g(w)=g1(w)+g2(w)+⋯+gn2(w)+gn2+1(w)+⋯+gn−1(w)+gn(w)

1-n/2是第一个处理器计算，计算结果记为g1~，n/2-n是第二个处理器,计算结果为g2~。

得到结果：g(w)=g1~+g2~

  • 单纯的并行计算不难，难在如何通信部分。

基于MapReduce的同步（Synchronous）并行梯度下降算法

MapReduce 是Google开发的一个编程模型和软件系统，

  • CS architecture

  • message-passing comunication

  • bulk synchronous parallel(批量关联同步)

MapReduce不开源，因此有了很多替代的版本。

比如Apache Hadoop(MapReduce的一个开源实现。)

还有现在的Apache Spark，是上一个提升版本。

MapReduce

多个work nodes 计算map 将计算结果传回给sever

  • Sever把新的模型wt广播给所有结点

  • worker用本地数据和wt计算梯度gi~

  • Reduce操作，worker将这些gi~传好以后，向量求和得到g

  • Sever进行梯度下降，给参数w进行更新。得到wt+1

这样就是一轮Mapreduce ，系统不断的迭代重复这些操作，直到算法收敛。

虽然每个人只做了1m份，但是时间并不是变成1m，因为要计算通信和并行时间。
