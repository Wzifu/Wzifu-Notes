法二：减而治之的方法 （分治法，减少问题规模）

主要是快排本身，尤其是408的快排，和算法基础里的，如果遇到了顺序数组或者逆序数组，会退化成链表，时间复杂度会成为O（N2),默认情况都是选择的是第一个或者固定值，算法本身对partition也有优化，这里采用的事随机初始化Pivot元素，为了避免极端用例。大佬写的真的很好，下面是大佬的视频出处。

快排的方法：

  1. 第一种划分方法：

  • 对于完全有序，或者逆序，递归树倾斜，时间复杂度o(N2)

  • 解决办法：随机切分元素，破坏数组的有序性

  • 产生的问题：随机选择切分元素对[有大量重复元素的数组]无效

  • 解决办法：

    1. 把pivot相等的元素平均的划分到数组的两侧（第二种划分方式双路快排）

    2. 把pivot相等的元素挤到数组的中间 （第三种划分方式三路快排）

  1. 循环不变量：

  • 我们写的这一段代码到底在做什么

  • 知道设计的变量的定义是什么，变量一旦定义，就需要始终保持和这个定义

法三：建立堆方法解决

建堆的思路是最基本的数据结构内容了，这里很多大佬讲的很好，我本人是基于本科数据结构的内容去改写的。看了一下官方的方法并不是太好，1.是并没有严格取到第一个非叶节点（这仍然可以实现，但是会多几轮判断，效率降低。）2.采用递归的方法，在调整时候，这个会明显的提高时间，而且可能会为是否不用递归的方法。这里进行改进。

除了大根堆也可以采用小根堆，但是小根堆的话就是选取k个的小根堆，每次都和剩下的比较，最后保证小根堆里是k个最大的，那么最后小根堆的顶部，nums[0]就是我们要得到的结果！

这么做有什么好处？用小根堆，有一种解释是数据不用一次全部调入内存。这看似很好，但是实际上在工程运用中，很少会有单机的场景直接装入内存。

  • 小根堆：

由于小根堆的写法，其实总体上和大根堆逻辑一样，主要是先建立长度为K的小根堆，原地建立，并且依次对后面的遍历，符合的交换进堆调整。

首先，代码创建了一个大小为k的最小堆，并将数组中前k个元素添加到堆中。然后，代码遍历数组中剩余的元素。对于每个元素，代码首先获取堆顶元素（即堆中最小的元素）。如果当前元素大于堆顶元素，则将堆顶元素弹出，并将当前元素加入堆中。这样，堆中始终维护着数组中前k大的元素。

最后，代码返回堆顶元素，即第k大的元素。

优先队列PriorityQueue

PriorityQueue 是 Java 中的一种数据结构，它实现了 Queue 接口。它是一个基于优先级的队列，其中元素按照它们的自然顺序或者根据传递给构造函数的比较器进行排序。

在这段代码中，PriorityQueue 被用来实现一个最小堆。构造函数接受两个参数：初始容量 k 和一个比较器 Comparator.comparingInt(a -> a)。这个比较器用来确定元素的顺序，它表示元素按照它们的自然顺序进行排序。

Comparator.comparingInt(a -> a) 是一个比较器，它用来确定元素的顺序。它是 Comparator 类的一个静态方法，接受一个函数作为参数。这个函数用来从元素中提取一个 int 值，然后根据这个值对元素进行排序。

在这段代码中，传递给 comparingInt 的函数是 a -> a，它是一个恒等函数，表示元素本身就是要比较的值。因此，这个比较器表示元素按照它们的自然顺序进行排序。

同时由于 这类采用Integer的类型的，引入了泛型。Integer是java的一个类，是整数类型的对象，是Int类型的包装类，基本数据int和Integer是不同的，基本数据类型存储在栈中，而引用类型存储在堆中，队列这种类，只能存储引用类型的元素。

关于拆箱和装箱

Integer 类型的对象可以直接与 int 类型的值进行比较。这是因为 Java 支持自动装箱和拆箱。

自动装箱是指将基本数据类型的值自动转换为对应的包装类对象。例如，当你将一个 int 类型的值赋给一个 Integer 类型的变量时，Java 会自动将这个 int 值转换为一个 Integer 对象。

自动拆箱是指将包装类对象自动转换为对应的基本数据类型的值。例如，当你将一个 Integer 对象与一个 int 值进行比较时，Java 会自动将这个 Integer 对象转换为一个 int 值。

因此，在这段代码中，当 Integer 类型的堆顶元素与 int 类型的数组元素进行比较时，Java 会自动将堆顶元素拆箱为一个 int 值，然后再进行比较。
