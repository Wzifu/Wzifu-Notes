法1. 一分为二 有序和无序区间

这就是参考的官方解答，虽然是数组本身不是完全有序，是局部有序，但是在此题目中，总可以采用二分的思想，一分为二，一半有序，一半不一定有序。所以这里就有一个思路，我们可以采用一分为二的思想：

  1. 找到mid，并判断mid对应的值和target大小，等于就返回，如果不是，进行下一步判断

  2. 假设当[0,...,mid-1]有序,[mid+1,...,length-1]无序，

    1. 在有序区间上，假设nums[0]<=target<nums[mid],说明结果可能在此区间，接下来就是二分的迭代 right=mid-1

    2. 假设nums[0]<=target<nums[mid]不成立，说明答案不会在这有序区间内，因此 left=mid+1

  3. 假设当[0,...,mid-1]无序,[mid+1,...,length-1]有序，

    1. 在有序区间上，假设nums[mid]<target<=nums[length-1],说明结果可能在此区间，接下来就是二分的迭代 left=mid+1

    2. 假设nums[0]<=target<nums[mid]不成立，说明答案不会在这有序区间内，因此 right=mid-1,继续迭代

核心就是每次都能分为一个有序和一个不一定有序的，如果范围可以确认在有序里面，那么就很好办了，不然就在另一端不一定有序的里面，在此找有序无序，依次迭代。

法2.两次二分，第一次找两段的边界，第二次对应的区间二分找值

之前也是看了三叶大佬的题解，对二分本质有了深刻理解

「二分」不是单纯指从有序数组中快速找某个数，这只是「二分」的一个应用。

「二分」的本质是两段性，并非单调性。只要一段满足某个性质，另外一段不满足某个性质，就可以用「二分」。

按照这个思想，由于数组可以分为两段，左和右，左区间的值都大于右区间，如[4,6,7,8,1,2,3]。因此可以去找分界点。到目前为止，其他题解中已经说的很明白，这个思路还是很明显，关键在于如何实现和细节，这里先不去看题解的解答，会有点乱。我们从这个思路一点点分析。

  • 要找边界，比如刚才的例子，你可以把8看做边界，1看做边界。8作为左区间的右边界，1作为右区间的左边界(这里非常重要，一定要记住，理解可以不同，但是如何区分的心里要有数)，在这里我找右区间的左边界，即1所在的位置。（另一个的代码在下面我也给出）

  • 那么如何在第一次循环中找到这个边界呢，而且这不是一个具体的值，我们以往的二分不是都在给定值的情况下去比较，不是找整体范围。那么这怎么用二分实现呢？

为了回答上面的问题，参考了一些书籍和网上的资料，有些当时没有标注（忘记了出处，根据记忆归纳，如果后面找到了会在最后补上）。总的来说二分查找还可以去查找左右边界。这里用的就是二分查找左边界的模型（具体最后会总结）。

利用二分法寻找左边界是二分查找的一个变体，应用它的题目常常有以下几种特性之一：

  1. 数组有序，但包含重复元素

  2. 数组部分有序，且不包含重复元素（本体就是这个情况）

  3. 数组部分有序，且包含重复元素

 

既然要寻找左边界，那我们应该是从右边开始搜索，不断地往左。即使我们找到了符合的条件，可以是nums[mid]==target，也可以是一个范围如nums[mid]<=target。但此时的mid位置也不一定就是最左侧的那个边界，我们需要根据情况去收缩右边界。

此时算法有一个模板：

  • 循环条件： left < right

  • 中间位置计算： mid = left + ((right -left) >> 1) (如果不怕越界可以直接left+right >> 1)

  • 左边界更新：left = mid + 1

  • 右边界更新： right = mid

  • 返回值： nums[left] == target ? left : -1

由于mid = left+((right-left))所以每次left其实是靠中间往左的位置（向下取整）。right = mid，因此不断往左靠拢寻找左边界。 这里的循环条件要注意了，不再是left<=right，而是left < right。这里可以举个例子，在left和right是邻居的时候，比如[1,3],left=1 right =3 mid =1;按照步骤走下去，下一步一定是left=mid=right;如果还采用leftM<=right，有两个情况：

  • 比如目标值target为1，nums[mid]>=target(1>=1),那么right=mid,会一直循环下去

  • 当目标值target为4，最后会left!=right，跳出。

因此不能统一，所以只取left<right.

所以我们只遍历到最后一轮时候，这一轮之后，无论如何,left=mid=right。

  • 如果这个位置的值，等于目标值，那么就一定是最左侧的目标值；

    • （因为之前讨论了，用这个方法可以找边界，是符合要求的边界值，因此才说一定是最左侧的目标值，比如重复元素，小于某个值这些区间的边界。同样也可以用来找单个目标值，这个值就是目标的边界，就是其本身。）

  • 如果不等于目标值，说明没有找到目标值。

 

理解了上述的思想，再来回到题目：

已经讲过了二分的本质在于二段性，数组本身分为了两端，也可以看做有序的，第一段的值都>=nums[0],第二段的值都<=nums[len] (这里我不太建议表示为 第二段的值都<nums[0]，毕竟第一段第二段应该各自分开嘛，和自己的比多好~，而且代码后面也表明这种逻辑上更好理解)，因此我们要用二分的方法找到分界点，你要知道，自己是想找那个段的边界。原题解中是求第一段的右边界（第一段的最后一个值），我在这里是按着我的思想来求第二段的左边界（第二段的第一个值）。按照这个思路走下去。

return的问题

return nums[left]==target ? left : -1; （为什么一定要是left 不能是right？）

由于我们第二个二分查找不一定执行，所以left不一定会等于right

  • 当执行了第二个二分时候，退出的结果一定是left=right,那么此时left和right值一样，但是我认为在此题意义本身不同

  • 当没有执行第二个二分时候，left要么是第一段的开头或者第二段的开头（左边界），所以肯定不会越界，但是right可能会为-1.所以输入他会越界。

所以最好养成个习惯，找左区间的时候，就用left进行返回确认，会避免很多问题。
